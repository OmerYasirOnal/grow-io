## What shipped in v1.3

A coherent release across all four pillars, turning grow.io from a single endless cosmetic-only arena
into a multi-mode game with a real progression spine, a fixed coin economy, and a premium feel.

**Progression / meta**
- Account **XP + Level** with a **MILESTONES** coin-reward track (every run, even a bad one, earns XP).
- Named **Score Tiers** Bronze ŌåÆ Silver ŌåÆ Gold ŌåÆ Platinum ŌåÆ Diamond ŌåÆ Master (badge on menu + death).
- **3 Daily Missions** (date-seeded; coins + XP; all-3 bonus) ŌĆö a tappable chip ŌåÆ panel on the menu.
- **10 Achievements** reading lifetime stats (a dedicated screen off the leaderboard).

**Modes**
- A `mode` / `MODES` foundation + a **Mode Select** screen.
- **Rush** (90-second score sprint) and **Boss Rush** (back-to-back escalating boss waves, scored by
  waves) on top of **Classic**; per-mode best scores.

**Economy / content**
- **Fixed the coin-economy leak** (score-tier skins are earn-only; kill/length/games skins are coins-only)
  so coins finally matter. Existing owned skins are grandfathered.
- A second cosmetic axis ŌĆö coins-only **Trails** (Sparkle / Embers / Neon Streak / Bubbles / Rainbow Dust).
- A weekly deterministic **Featured** shop slot (discounted).
- Better rewarded-ad value: **"Double your coins"** on the death screen + a scaling menu reward + a
  **7-day Daily Reward** modal with an ad-funded streak-save.

**Feel / accessibility**
- **Calm ambient pad music** (replaced the old 750 ms blip loop).
- Premium juice: **shockwave + additive-glow** on kills/death/boss, **visceral boost** (head glow +
  speed-lines), **living eyes / eat punch / boundary danger glow**, **zones ŌåÆ biomes** (gradient bg).
- A **Reduced-Motion** setting that gates the heavy additive VFX.

## How it was built & verified

Built autonomously via deterministic **Workflow** orchestration: each feature = a build agent (green-gated:
typecheck + jest + a `node --check` parse of the game script) ŌåÆ a 3-lens adversarial verify (correctness /
blank-the-game risk / regression) ŌåÆ a blocker-fix pass. Then **two full read-only audits** of the committed
code, each followed by a fix workflow. The controller browser-smoked every feature (Playwright) and ran the
migration tests below.

**Two real audit bugs were caught and fixed:**
1. `loadBlob(ns, {})` with empty defaults dropped all keys ŌåÆ per-mode bests never persisted. Fixed (whole-object case).
2. **(HIGH)** The native STATS-blob "floor" adopted a cross-device backup into `gBlobs` but never re-hydrated
   the in-memory game vars, so the first gameplay write clobbered a restored save. Fixed via a single
   `hydrateBlobState()` called at boot **and** after the STATS_LOADED floor-adoption. Plus: a `_parseHex`
   `||255` bug that mis-tinted the default skin's boost glow, a `statsReconciled` deferral mirroring the coins
   path, and the Boss-Rush "best" wave-suffix.

**Save-migration proven safe (in-browser, both scenarios):**
- *v1.2 ŌåÆ v1.3 update:* coins / high-score / owned skins / stats all preserved; v1.3 state initializes fresh.
- *Cross-device / cleared-cache restore:* the native blob floor rehydrates XP / bests / trails / achievements
  with no first-write corruption.
  *(A real-device confirmation on the first archive is still good practice ŌĆö see below.)*
