# grow.io v1.3 "Worlds & Progression" Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to
> implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v1.3 — a progression spine (XP/levels/missions), a fixed coin economy, new modes
(Rush, later Boss-Rush) on a mode-config foundation, and premium arena juice + cosmetic trails —
on the live single-file Canvas2D game, without corrupting existing saves or blanking the game.

**Architecture:** All gameplay lives in one `'use strict'` classic script `game/index.html`
(loaded as a file:// asset, so NO ES modules). A thin Expo/RN shell (`src/`) bridges ads/haptics/
AsyncStorage over `postMessage`. v1.3 adds systems INLINE, each in its own `// ═══ NAME ═══` banner
block, persisted through one new namespaced helper that piggybacks the existing STATS native bridge.

**Tech Stack:** Vanilla JS + Canvas2D (game), TypeScript + Jest (src/ shell), Expo/EAS (build).

**Spec:** `docs/2026-06-02-v1.3-worlds-progression-design.md`.

## Conventions & definition of "green" (read first — applies to every task)

- **Single-file, strict mode.** Every new global declared with `let`/`const` (a bare assignment is a
  ReferenceError that blanks the game). Top-level `function` decls are fine (they land on `window`).
- **Section contract.** Each new system = its own `// ═══ NAME ═══` block, in the existing order:
  state/buffers near the globals (~:152), an `update*(dt)` called from `update()`, a `draw*()` wired
  into the render switchboard (:1224-1254), save/load in the persistence block (:355).
- **No per-frame allocation** in `update`/`render`. Reuse module buffers (`liveBotsBuf` :1219 +
  `.length=0`/refill). Cache canvases (gradients) and regenerate only on settle.
- **No new native message types.** New persisted data piggybacks the **STATS** payload (Task 0.1).
- **"Green" per PR (no CI in repo):**
  1. `npm run typecheck` → clean.
  2. `npm test` → all Jest pass.
  3. Browser smoke: `npx http-server game -p 8091 -c-1`, load `http://127.0.0.1:8091/index.html`,
     dismiss the tutorial, and click through the affected screen — **zero JS console errors**
     (favicon 404 is the only allowed error). A JS error halts the whole script, so
     "functions defined + screen renders clean" is a strong signal.
- **Loop per feature:** GitHub issue → sub-branch off `feat/v1.3-worlds-progression` → implement →
  green → PR into the integration branch → fresh zero-context adversarial review → merge. Smoke
  `menu→play→death→menu` after each merge before stacking the next.
- **Branch:** integration branch `feat/v1.3-worlds-progression` (already created, holds the spec).
- **Public release:** owner does the final Xcode archive → Distribute. This plan gets a testable
  build to that point.

### Task 0.1: Versioned namespaced save helper (`saveBlob`/`loadBlob`) piggybacking STATS

**Design:** All v1.3 persisted data lives under ONE authoritative localStorage key `growio_v13`
holding `{v:1, <ns>:{...}, ...}` (namespaces: `prog` XP/level, `missions`, `bests` per-mode,
`cos` cosmetics/trails, `daily` claim-state). The native AsyncStorage mirror is carried inside the
existing STATS payload as a new `blob` field (a first-load floor only — never lowers local), so
**no new native message type** ships.

  `saveStats()` (:366).
- Modify (shell): `src/storage.ts` (widen `Stats` with an optional `blob`), round-trip in
  `saveStats`/`loadStats`.
- Modify (shell): `src/GameWebView.tsx` STATS save/load handlers (pass `blob` through).
- Test: `src/storage.test.ts` (Jest — blob round-trip / default / corrupt).

```ts
export interface Stats { totalKills: number; totalGames: number; totalDeaths: number; blob?: Record<string, unknown>; }

export async function loadStats(): Promise<Stats> {
  try {
    const raw = await AsyncStorage.getItem(STATS_KEY);
    if (!raw) return { totalKills: 0, totalGames: 0, totalDeaths: 0 };
    const parsed = JSON.parse(raw);
    const out: Stats = {
      totalKills: typeof parsed.totalKills === 'number' ? parsed.totalKills : 0,
      totalGames: typeof parsed.totalGames === 'number' ? parsed.totalGames : 0,
      totalDeaths: typeof parsed.totalDeaths === 'number' ? parsed.totalDeaths : 0,
    };
    if (parsed.blob && typeof parsed.blob === 'object' && !Array.isArray(parsed.blob)) out.blob = parsed.blob;
    return out;
  } catch { return { totalKills: 0, totalGames: 0, totalDeaths: 0 }; }
}

export async function saveStats(stats: Stats): Promise<void> {
  try { await AsyncStorage.setItem(STATS_KEY, JSON.stringify(stats)); } catch {}
}
```

```js
// ═══ V1.3 SAVE BLOBS (namespaced, versioned, additive) ═══
// One authoritative localStorage key holds every v1.3 namespace; the native
// mirror rides inside the STATS payload (blob field) as a first-load floor only.
const V13_KEY='growio_v13', V13_VER=1;
function loadBlobs(){try{const d=localStorage.getItem(V13_KEY);if(d){const o=JSON.parse(d);if(o&&typeof o==='object')gBlobs=o;}}catch(e){}if(typeof gBlobs!=='object'||!gBlobs)gBlobs={};gBlobs.v=V13_VER;}
function persistBlobs(){try{localStorage.setItem(V13_KEY,JSON.stringify(gBlobs));}catch(e){}saveStats();} // saveStats now also mirrors gBlobs to native
function loadBlob(ns,defaults){const cur=(gBlobs&&typeof gBlobs[ns]==='object')?gBlobs[ns]:null;const out={};for(const k in defaults)out[k]=(cur&&k in cur)?cur[k]:defaults[k];return out;}
function saveBlob(ns,obj){gBlobs[ns]=obj;persistBlobs();}
```

### Task 1.1: Account XP + Level + MILESTONES
- **Hooks:** award XP in the death/scoring block (`killSnake` :902) from score+kills+length; a
  `MILESTONES` table (level→reward: coins / exclusive cosmetic / nothing); persist
  `saveBlob('prog',{xp,lvl})`. Render a thin XP bar on the menu + a level-up toast (reuse
  `showToast`/`drawToasts` :449/:454).
- **Acceptance:** every run (even a 200-score death) grants XP; crossing a level fires the toast +
  grants the milestone reward exactly once; XP/level survive reload; classic flow unbroken.
- **Verify:** browser — play a short run, die, confirm XP increments + persists; typecheck/test green.

### Task 1.2: Decouple the dual-unlock economy leak + retune prices
- **Hooks:** `checkUnlocks` (:1040) — feat skins (`u` of type `k`/`l`/`g`/score-tier reserved as
  "earn") become EARN-ONLY (auto-unlock, **no Buy button**); the rest become coins-only; gate the
  shop Buy path (:2231) on a `buyable(sk)` predicate; retune `skinPrice` (:417) to a real 2-4 week
  curve. **Migration-safe:** owned indices preserved (`loadSkins` filter :362); grandfather earned
  skins.
- **Acceptance:** a player can no longer get a coins-priced skin for free by hitting a score
  threshold; already-owned skins stay owned across the update; coins now have a binding sink.
- **Verify:** browser — confirm a locked coins-only skin shows a price and a Buy gated on coins;
  confirm an existing `unlockedSkins` set is unchanged after load. Add/keep a note in the PR on the
  retuned curve.

### Task 1.3: Shockwave + additive-glow particles on kill/death/boss
- **Hooks:** new pool beside `spawnP` (:505-538) with a reused buffer; fire from `killSnake`
  (:882/:900); draw with `ctx.globalCompositeOperation='lighter'` **bounded to the effects layer**
  (save/restore around it). Radial expanding ring + color-graded burst.
- **Acceptance:** a kill/death reads as a distinct "pop" (not the flat eat-dot); no per-frame
  allocation; FPS stable with 12 bots; additive pass does not bleed into snake/food rendering.
- **Verify:** browser — trigger a kill, observe shockwave; check console clean; eyeball perf.

### Task 1.4: Rush mode (90s sprint) + per-mode best scores
- **Hooks:** add `rush` to `MODES` (`timeLimit:90`); in `update()` win/lose (:1136) end the run when
  `playTime≥timeLimit` (killerName `'time'`); HUD timer in `drawHUD` (:1442); override food/pu spawn
  cadence per mode (:1168-1169). Per-mode bests via `saveBlob('bests',{classic,rush})` — **extends,
  never replaces** `growio_hs`/`growio_top`. Death screen + leaderboard show a mode badge and branch
  the new-best check (:1908) per mode.
- **Acceptance:** Rush is selectable from modeselect, runs 90s with a visible countdown, ends with a
  score, records a separate best; classic best untouched.
- **Verify:** browser — play Rush to timeout, confirm end + separate best persists.

## Risks (from spec §6 — enforce per task)

Single-JS-error blast radius → strict globals + section contract + per-PR smoke. Save corruption →
everything through `saveBlob` + STATS-floor + on-device upgrade test (Wave 0 gate). Per-frame
alloc/fillrate → reused buffers + cached gradients + bounded additive pass. Scope creep → P0 is the
shippable core; P1/P2 cut-able. Economy backlash → grandfather owned skins, frame as "new to spend
on." Ad fatigue → all new rewarded opt-in/contextual, never gate progression behind an ad.
